## 4. เฟส 1 — Digital Clinic (สัปดาห์ที่ 1–3)

> **เป้าหมาย:** แทนที่กระดาษ คุณหมอสามารถเริ่มใช้แอปจัดการผู้ป่วยได้ทันที

### 4.1 ระบบยืนยันตัวตนและความปลอดภัย
- [ ] เชื่อมต่อ Supabase Auth (อีเมล/รหัสผ่าน)
- [ ] หน้าล็อก PIN (ตั้ง PIN 4–6 หลักในตั้งค่า)
- [ ] ยืนยันตัวตนด้วย Biometric (สแกนนิ้ว/Face ID ผ่าน `local_auth`)
- [ ] ระบบควบคุมสิทธิ์ตามบทบาท
  - `OWNER` / `DOCTOR` → เข้าถึงข้อมูลทั้งหมด
  - `RECEPTIONIST` → เข้าถึงจำกัด (ไม่เห็นข้อมูลการเงิน, ไม่เห็นประวัติการแพทย์เต็ม)
- [ ] ล็อกอัตโนมัติเมื่อแอปไปอยู่เบื้องหลัง (ตั้งเวลาได้)
- [ ] หน้านโยบายความเป็นส่วนตัวในแอป (ตาม PDPA)

### 4.2 หน้าแดชบอร์ด
- [ ] ส่วนหัวทักทายพร้อมชื่อหมอ + วันที่ปัจจุบัน
- [ ] การ์ดสถิติ 3 ใบ: นัดหมายวันนี้, ผู้ป่วยทั้งหมด, ติดตามผล
- [ ] รายการนัดหมายวันนี้ (เรียงตามเวลา)
  - ชื่อ + ชื่อเล่นผู้ป่วย
  - ประเภทหัตถการ
  - แบดจ์สถานะ (ใหม่ / ยืนยัน / ติดตามผล)
  - แบดจ์ VIP / ⭐⭐⭐
  - ปุ่มส่งข้อความ LINE/WhatsApp ต่อนัดหมาย
- [ ] กริดปุ่มลัด (4 ปุ่ม):
  - เพิ่มผู้ป่วยใหม่
  - นัดหมายใหม่
  - อัปโหลดรูป
  - วาดไดอะแกรม

### 4.3 จัดการผู้ป่วย
- [ ] หน้ารายชื่อผู้ป่วย พร้อมค้นหา (ชื่อ, ชื่อเล่น, HN)
- [ ] การ์ดผู้ป่วย แสดง: อักษรย่อ, ชื่อ, ชื่อเล่น, อายุ, เบอร์โทร, มาล่าสุด, แบดจ์ช่องทาง
- [ ] ตัวบ่งชี้สถานะผู้ป่วย: VIP (ทอง), ⭐⭐⭐ (แดง), ปกติ
- [ ] ปุ่ม FAB → เพิ่มผู้ป่วยใหม่ (กรอกเอง หรืออนาคต OCR สแกน)
- [ ] **โปรไฟล์ผู้ป่วย** (overlay เต็มจอ, เลื่อนเข้ามา):
  - ส่วนหัว: รูปโปรไฟล์, ชื่อ, ชื่อเล่น, อายุ, HN, แบดจ์ VIP/star
  - ปุ่มส่งข้อความ (LINE / WhatsApp ตามช่องทางของผู้ป่วย)
  - ปุ่มนัดหมาย
  - **แท็บเลื่อนแนวนอน:**
    1. **ข้อมูล:** ข้อมูลส่วนตัว, เลือกสถานะผู้ป่วย, เอกสารประจำตัว
    2. **HA (ประเมินสุขภาพ):** ยาที่แพ้, โรคประจำตัว, สูบบุหรี่/ดื่มแอลกอฮอล์
    3. **💉 ฉีด:** ประวัติหัตถการฉีด, การ์ดขยายได้
    4. **⚡ เลเซอร์:** ประวัติหัตถการเลเซอร์
    5. **🩺 ทรีตเมนต์:** ประวัติทรีตเมนต์อื่นๆ
    6. **+ บันทึก:** ฟอร์มบันทึกการรักษา (เช็คเตือนก่อนเปิดฟอร์ม)
    7. **รูปภาพ:** คู่รูป Before/After + หน้าเปรียบเทียบ
    8. **✂️ ศัลยกรรม:** ประวัติศัลยกรรม พร้อมชิปช่วงเวลา
    9. **💰 ค่าใช้จ่าย:** สรุปการเงิน, คอร์ส, ประวัติชำระเงิน

### 4.4 บันทึกการรักษา (ฟอร์ม SOAP — แบบไหลลื่น ไม่ใช่ช่องตาราง)
> ตามคำขอลูกค้า: "แต่ไม่ต้องทำเป็นช่องๆแบบนี้" — แบบไหลลื่น ไม่ใช่กริดช่อง

- [ ] ส่วนหัว: วันที่ (อัตโนมัติ) + ชื่อหมอ (อัตโนมัติจากตารางเวร)
- [ ] **ส่วน: สัญญาณชีพ (Vitals)** — กริดช่องกรอก:
  - อุณหภูมิ (°C), ชีพจร (/นาที), ความดัน (ค่าบน/ค่าล่าง mmHg), SpO₂ (%)
  - คะแนนปวด NRS (ปุ่มกด 0–10)
- [ ] **ส่วน: อาการสำคัญ (CC)** — ช่องข้อความอิสระ
- [ ] **ส่วน: การตรวจร่างกาย (Objective / PE)** — ช่องข้อความอิสระ
- [ ] **ส่วน: การวินิจฉัย (Assessment / DX)** — ช่องข้อความอิสระ
- [ ] **ส่วน: แผนการรักษา (Plan)** — ช่องข้อความอิสระ
- [ ] **ส่วน: ผลิตภัณฑ์ที่ใช้** — overlay เลือกสินค้า:
  - แท็บหมวดหมู่ (Botox, Filler, Biostimulator, Polynucleotide, Skinbooster, Laser, อื่นๆ)
  - รายการสินค้า พร้อมชื่อ, ยี่ห้อ, แบดจ์หน่วย
  - กรอกปริมาณ + เลือกหน่วย
  - สินค้าที่เลือกแล้วแสดงเป็นแท็กลบได้
- [ ] **ส่วน: ไดอะแกรมบนใบหน้า (ย่อ)** — canvas ฝังในฟอร์ม:
  - เครื่องมือ: ✏️ วาด, 📍 หมุด, ↩️ ย้อนกลับ, 🗑️ ล้าง
  - จานสี + เลือกขนาดปากกา
  - เทมเพลตรูปใบหน้า (ด้านหน้า)
  - ป้ายกำกับหมุด (เช่น "20U", "1ml") วางบน canvas
  - ตัวบ่งชี้โหมด ("โหมดดู" / "โหมดวาด")
- [ ] **ส่วน: บันทึกการรักษา / พารามิเตอร์เลเซอร์:**
  - เครื่อง, พลังงาน, Pulse/Spot, จำนวน Shots ทั้งหมด
- [ ] **ส่วน: Progress Note:**
  - ผลตอบสนองต่อการรักษาครั้งก่อน: ชิป (ดีขึ้น / คงที่ / แย่ลง)
  - ผลข้างเคียง: ชิปเปิด/ปิด (ไม่มี, ผิวแดง, ไหม้, PIH, บวม)
- [ ] **ส่วน: คำแนะนำและนัดติดตาม:**
  - เช็คลิสต์ (หลีกเลี่ยงแดด, ทากันแดด SPF30+, ทายาตามสั่ง)
  - นัดครั้งต่อไป: เลือกวัน + เวลา
- [ ] **ปุ่มบันทึก** → ตรวจสอบ, บันทึกลง `treatment_records`, หักสต็อก, สร้างนัดหมายถ้าตั้งนัดติดตาม

### 4.5 วาดไดอะแกรมบนใบหน้า (ฟีเจอร์เต็ม)
- [ ] **เอ็นจิ้นวาด canvas** ใช้ `CustomPainter` + จับท่าทางสัมผัส
- [ ] เครื่องมือ: วาดอิสระ, หมุด (pin + ป้าย), ยางลบ, ย้อนกลับ, ล้าง
- [ ] สลับมุมมอง: ด้านหน้า / ด้านข้าง / โซนปาก
- [ ] จานสี: แดง, น้ำเงิน, เขียว, ดำ, ส้ม, ม่วง
- [ ] เลือกขนาดปากกา: บาง (2px), กลาง (4px), หนา (8px)
- [ ] ป๊อปอัพหมุด: แตะตำแหน่ง → กรอกป้าย → ยืนยัน/ยกเลิก
- [ ] ตัวบ่งชี้ซูม (บีบซูมบน iPad)
- [ ] **บันทึกขึ้นคลาวด์** ทันที (Supabase Storage)
- [ ] โหลดไดอะแกรมก่อนหน้าของผู้ป่วย
- [ ] ป้องกันการเลื่อนจอขณะวาด (touch-action: none)

### 4.6 กระดาษโน้ตดิจิทัล (หน้าว่าง)
- [ ] หน้า canvas ว่างเปล่า เข้าถึงได้จากบันทึกการรักษา
- [ ] วาดอิสระด้วยนิ้ว/สไตลัส
- [ ] บันทึกเป็นรูป → แนบกับเวชระเบียนผู้ป่วย
- [ ] รองรับ ย้อนกลับ/ทำซ้ำ

### 4.7 ใบยินยอม + ลายเซ็นดิจิทัล
- [ ] จัดการเทมเพลต (แอดมินสร้างแม่แบบใบยินยอมต่อหัตถการ)
- [ ] แสดงใบยินยอม (เนื้อหาเลื่อนอ่านได้)
- [ ] **แผ่นลายเซ็นดิจิทัล** — ผู้ป่วยเซ็นด้วยนิ้วบนหน้าจอ
- [ ] จับภาพลายเซ็น → เก็บใน Supabase Storage
- [ ] บันทึกใบยินยอม พร้อม: ผู้ป่วย, หัตถการ, ลายเซ็น, เวลา, พยาน
- [ ] อนาคต: สร้าง PDF สำหรับพิมพ์/เก็บ

### 4.8 ปฏิทินและนัดหมาย
- [ ] ปฏิทินแบบรายเดือน พร้อมจุดนัดหมาย
- [ ] เลื่อนเดือน: ก่อนหน้า/ถัดไป
- [ ] มุมมองรายวัน: รายการนัดหมายของวันที่เลือก
- [ ] แต่ละนัดแสดง: เวลา, ชื่อ+ชื่อเล่นผู้ป่วย, หัตถการ, แบดจ์สถานะ
- [ ] **ตารางเวรพนักงาน:**
  - ตั้งว่าหมอคนไหนเข้าเวรวันไหน
  - ทำเครื่องหมาย: เข้าเวร / ลา / ครึ่งวัน
  - ตอนสร้างนัดหมาย → ระบบเช็คว่าหมอว่างไหม
  - ตอนบันทึกการรักษา → กำหนดหมอจากตารางเวรอัตโนมัติ

### 4.9 ระบบรูป Before/After
- [ ] **อัปโหลดรูป** ผ่านกล้องหรือแกลเลอรี (บีบอัดเพื่อประหยัดพื้นที่)
- [ ] จัดหมวด: ก่อนทำ, หลังทำ (1 เดือน, 3 เดือน, 6 เดือน), ติดตามผล
- [ ] **เลย์เอาต์เปรียบเทียบ 4 ช่องแนวนอน** (แนวนอนบน iPad):
  - ช่อง 1: Before (ซ้ายสุด)
  - ช่อง 2–4: After (1 เดือน, 3 เดือน, 6 เดือน)
- [ ] **ซูมและแพนพร้อมกัน** (สำคัญมาก):
  - บีบซูมรูปไหน → ทั้ง 4 รูปซูมไปจุดเดียวกัน
  - เลื่อนรูปไหน → ทั้ง 4 รูปเลื่อนตาม
- [ ] **ปรับตำแหน่งด้วยมือ:**
  - ลากแต่ละรูปเพื่อปรับตำแหน่งในกรอบ
  - หมุนรูปในกรอบเพื่อจัดแนวใบหน้า
- [ ] **ส่งออกภาพรวมคลิกเดียว:**
  - รวม 4 รูปเป็นรูปเดียวแนวนอน
  - ใส่อัตโนมัติ: โลโก้คลินิก + วันที่/ป้ายกำกับใต้แต่ละรูป
  - ส่งออกเพื่อแชร์ผ่าน LINE/WhatsApp
- [ ] **ประสิทธิภาพ (ปรับแต่งสำหรับ iPad):**
  - แคชรูปในเครื่อง (โหลดทันทีเมื่อเปิดซ้ำ)
  - ลดขนาดรูปสำหรับแสดง 4 ช่อง (ป้องกัน RAM ล้น)
  - โหลดรูปย่อล่วงหน้าในแกลเลอรี
- [ ] ฟิลด์ฐานข้อมูล: `image_type`, `treatment_date`, `clinic_id`

## 5. เฟส 2 — Business Logic (สัปดาห์ที่ 3–5)

> **เป้าหมาย:** คอร์ส, การเงิน, ราคา, สต็อก — เครื่องยนต์ธุรกิจ

### 5.1 ระบบคอร์ส
- [ ] **สร้างคอร์ส:**
  - ชื่อคอร์ส, เชื่อมบริการ, ราคา
  - จำนวนครั้งที่ซื้อ (X) + จำนวนครั้งแถม (Y) → รวม = X + Y
  - วันหมดอายุ
- [ ] **แสดงการ์ดคอร์ส:**
  - ชื่อ, ข้อมูลซื้อ/แถม, หมดอายุ, ราคา
  - แบดจ์สถานะ: ใช้งาน (เขียว), เหลือน้อย (ทอง), ครบ (เทา), หมดอายุ (แดง)
  - **กริดจุดครั้ง (Session dots):**
    - วงกลมมีเลขสำหรับครั้งที่ซื้อ
    - 🎁 วงกลมสีทองสำหรับครั้งแถม
    - แตะเพื่อสลับใช้แล้ว/ยังไม่ใช้
  - แถบความคืบหน้าไล่สี
  - จำนวนคงเหลือ
- [ ] **ตารางภาพรวมคอร์ส** — หน้าเดียวแสดงคอร์สของผู้ป่วยทุกคน:
  - กรองได้ตามสถานะ, ผู้ป่วย, ประเภทคอร์ส
  - คอลัมน์: ผู้ป่วย, คอร์ส, ใช้แล้ว/ทั้งหมด, สถานะ, หมดอายุ
- [ ] เชื่อมการใช้ครั้งคอร์สกับบันทึกการรักษา

### 5.2 ระบบการเงิน
- [ ] **การ์ดสรุปค่าใช้จ่ายผู้ป่วย** (ส่วนหัวสีเข้มไล่สี):
  - ยอดใช้จ่ายสะสมรวม
  - จำนวนคอร์สที่ใช้งานอยู่
  - ครั้งคงเหลือ
  - ยอดค้างชำระ (เน้นสีแดง/terra ถ้า > 0)
- [ ] **จัดการยอดค้างชำระ:**
  - รายการยอดค้างต่อหัตถการ
  - ปุ่ม "บันทึกการชำระ" ต่อรายการ
  - รองรับชำระบางส่วน
- [ ] **บันทึกการชำระเงิน:**
  - จำนวนเงิน, วิธีการ (เงินสด, โอน, บัตรเครดิต, บัตรเดบิต)
  - เชื่อมกับหัตถการหรือคอร์ส
  - รายละเอียด/หมายเหตุ
- [ ] **ประวัติการชำระเงิน:**
  - รายการเรียงตามเวลา พร้อมไอคอน, ชื่อหัตถการ, วันที่, วิธีการ, จำนวนเงิน
  - กรองตามช่วงวันที่ได้

### 5.3 จัดการราคา (Price List — ในตั้งค่า)
- [ ] **ราคาเริ่มต้นต่อบริการ/สินค้า** แก้ไขได้ในตั้งค่า
- [ ] ราคาเติมอัตโนมัติในฟอร์มชำระเงิน
- [ ] การอัปเดตมีผลกับรายการใหม่เท่านั้น (ไม่ย้อนหลัง)
- [ ] ราคาต่อบริการ: ชื่อ, หมวด, ราคาเริ่มต้น
- [ ] ราคาต่อสินค้า: ชื่อ, ยี่ห้อ, หน่วย, ราคาเริ่มต้น

### 5.4 ควบคุมสต็อก
- [ ] **คลังสินค้า** (CRUD ในตั้งค่า):
  - หมวดหมู่: Botox, Filler, Biostimulator, Polynucleotide, Skinbooster, Laser, อื่นๆ
  - ฟิลด์: ชื่อ, ยี่ห้อ, หน่วย, จำนวนสต็อก, จำนวนต่อภาชนะ, วันหมดอายุ
  - เพิ่ม/แก้ไข/ลบสินค้า
- [ ] **แปลงหน่วย / สต็อกทศนิยม:**
  - ติดตาม vial ที่เปิดแล้ว: เช่น Toxin 1 ขวด = 50U → ใช้ 20U → แสดง 30U คงเหลือ
  - สต็อกทศนิยม (DECIMAL(12,4))
  - หักสต็อกอัตโนมัติเมื่อบันทึกการรักษาพร้อมสินค้า
- [ ] **ติดตามวันหมดอายุ:**
  - วันหมดอายุต่อสินค้า
  - วันหมดอายุระดับ batch ผ่าน `inventory_transactions`
  - แจ้งเตือนภาพเมื่อสินค้าใกล้/เลยวันหมดอายุ
- [ ] **แจ้งเตือนสต็อกต่ำ:**
  - เกณฑ์ `min_stock_alert` ต่อสินค้า
  - แจ้งเตือนบนแดชบอร์ดเมื่อต่ำกว่าเกณฑ์
- [ ] **บันทึกธุรกรรมสต็อก:**
  - รับเข้า (ซื้อ), ใช้ (การรักษา), สูญเสีย, ปรับปรุง

## 6. เฟส 3 — Smart System (สัปดาห์ที่ 5–7)

> **เป้าหมาย:** ระบบเตือนอัจฉริยะ, ส่งข้อความ, push notifications

### 6.1 ระบบเตือนก่อนทำหัตถการ
- [ ] **เช็คอัตโนมัติก่อนเปิดฟอร์มบันทึกการรักษา:**
  - เช็คประวัติหัตถการว่าระยะห่างใกล้เกินไปหรือไม่
  - เช็คข้อห้ามใช้ (retinoids, ยาต้านการแข็งตัวของเลือด)
  - เช็คยาที่แพ้กับสินค้าที่จะใช้
- [ ] **หน้าต่างเตือน (bottom sheet):**
  - ส่วนหัว: ไอคอน ⚠️ + "ตรวจสอบก่อนบันทึก"
  - รายการเตือนพร้อมระดับความรุนแรง:
    - 🚫 **หยุด (แดง):** ระยะห่างหัตถการใกล้เกินไป (เช่น Filler < 120 วัน)
    - ⏱️ **ระวัง (ทอง):** ต่ำกว่าระยะห่างที่แนะนำ
    - 💊 **ระวัง (ทอง):** ตรงกับยาที่แพ้
    - ⚠️ **ระวัง (ทอง):** ใช้ยาต้านการแข็งตัวของเลือด (เสี่ยงช้ำ)
    - 🚫 **หยุด (แดง):** ใช้ retinoids (ข้อห้ามสำหรับเลเซอร์)
    - 📋 **ข้อมูล (เขียว):** ลำดับการรักษาที่แนะนำ
  - แต่ละรายการ: ไอคอน, หัวข้อ, คำอธิบาย, แบดจ์ระดับ
  - ปุ่มดำเนินการต่อ (เปลี่ยนสีตามระดับความรุนแรง)
  - ปุ่มยกเลิก
- [ ] **กฎระยะห่างหัตถการ (ตั้งค่าได้):**
  ```
  Botox:   ขั้นต่ำ=60 วัน,  แนะนำ=90 วัน
  Filler:  ขั้นต่ำ=120 วัน, แนะนำ=180 วัน
  Laser:   ขั้นต่ำ=21 วัน,  แนะนำ=30 วัน
  HIFU:    ขั้นต่ำ=90 วัน,  แนะนำ=180 วัน
  ```

### 6.2 ส่งข้อความ LINE / WhatsApp
- [ ] **ช่องทางต่อผู้ป่วย** — แต่ละคนมี LINE ID และ/หรือเบอร์ WhatsApp
- [ ] **เทมเพลตข้อความ:**
  1. 🔔 แจ้งเตือนนัดหมาย (ชื่อ, วันที่, เวลา, หัตถการ)
  2. ✅ ยืนยันนัดหมาย
  3. 🌸 คำแนะนำดูแลหลังทำ (ตามประเภทหัตถการ)
  4. 📣 โปรโมชัน (แก้ไขได้)
- [ ] **หน้าเขียนข้อความ (bottom sheet):**
  - เลือกเทมเพลต (แตะเพื่อเลือก)
  - เติมชื่อผู้ป่วย, หัตถการ, วัน/เวลาอัตโนมัติ
  - ช่องแก้ไขข้อความ
  - ตัวอย่างข้อความ (เขียวสำหรับ LINE, เขียวเข้มสำหรับ WhatsApp)
  - ปุ่มส่ง (สีตามช่องทาง)
- [ ] **วิธีส่ง:**
  - LINE: Deep link ไปแอป LINE พร้อมข้อความ (`line://msg/text/...`)
  - WhatsApp: Deep link ไปแอป WhatsApp พร้อมข้อความ (`https://wa.me/...?text=...`)
  - อนาคต: LINE OA API / WhatsApp Business API สำหรับส่ง push
- [ ] **บันทึกข้อความ:** ติดตามข้อความที่ส่งทั้งหมดต่อผู้ป่วย

### 6.3 Push Notifications
- [ ] เชื่อมต่อ Firebase Cloud Messaging (FCM)
- [ ] แจ้งเตือนนัดหมาย → push ไปอุปกรณ์ของหมอ
- [ ] ตั้งเวลาแจ้งเตือนได้ (เช่น 1 วันก่อน, 1 ชั่วโมงก่อน)
- [ ] แจ้งเตือนสต็อกต่ำ
- [ ] แจ้งเตือนยอดค้างชำระ

### 6.4 สถานะซิงค์ออฟไลน์
- [ ] **ไอคอนสถานะซิงค์** ใน app bar:
  - ✅ "ซิงค์แล้ว" (เขียว) — ข้อมูลอัปเดตล่าสุด
  - 🔄 "กำลังซิงค์..." (เหลือง/หมุน) — กำลังอัปโหลด
  - ⚠️ "รอการเชื่อมต่อ" (แดง) — ออฟไลน์, คิวรอส่ง
- [ ] **สถาปัตยกรรม Offline-first:**
  - แคชข้อมูลในเครื่องด้วย SQLite ผ่าน `drift` หรือ `sqflite`
  - คิวบันทึกข้อมูลเมื่อออฟไลน์
  - ซิงค์อัตโนมัติเมื่อเชื่อมต่อกลับมา
  - หมอเห็นการรับประกันว่าไดอะแกรม/รูปภาพจะไม่หาย
