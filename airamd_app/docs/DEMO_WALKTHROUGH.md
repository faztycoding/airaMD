# แนวทางการเดโมระบบ

## เป้าหมาย

ใช้เอกสารนี้เป็นลำดับการนำเสนอ `airaMD` ให้เห็นว่าเป็นแพลตฟอร์มบริหารงานคลินิกที่พร้อมใช้งาน มีความเรียบร้อย และสอดคล้องกับบรีฟลูกค้าปัจจุบัน

## ลำดับการเดโมที่แนะนำ

### 1. ภาพรวม Dashboard
- เปิดหน้า `Dashboard`
- แสดงภาพรวมของวันนี้ ตัวชี้วัดการดำเนินงานที่มองเห็นได้ และแผงนัดหมายแบบเรียลไทม์
- เน้นว่าข้อมูลทางการเงินจะแสดงตามสิทธิ์ของผู้ใช้งาน

### 2. การลงทะเบียนผู้รับบริการและการตรวจสอบข้อมูล
- เปิดหน้า `Patients`
- สร้างผู้รับบริการใหม่
- สาธิต:
  - ฟิลด์ที่จำเป็นต้องกรอก
  - การตรวจสอบหมายเลขโทรศัพท์
  - การบังคับกรอกเลขบัตรประชาชนหรือ Passport
  - ช่องทางการติดต่อที่ต้องการ
- บันทึกและแสดงข้อความสำเร็จแบบพรีเมียม

### 3. Workflow การนัดหมาย
- เปิดหน้า `Calendar`
- สร้างนัดหมายใหม่
- สาธิต:
  - การเลือกวันและเวลา
  - การเลือกแพทย์ผู้รับผิดชอบ
  - ประเภทหัตถการ
  - สถานะนัดหมาย
- บันทึกนัดหมายและชี้ให้เห็นว่าชื่อแพทย์ผู้รับผิดชอบแสดงบนการ์ดนัดหมายแล้ว

### 4. Workflow การรักษาจากนัดหมาย
- เปิดนัดหมายที่เพิ่งสร้าง
- ใช้ปุ่ม `Open treatment record from this appointment`
- สาธิต:
  - ระบบแนะนำแพทย์อัตโนมัติจากนัดหมาย
  - ฟอร์ม SOAP ของการรักษา
  - การตรวจสอบการใช้สินค้า/สต็อก
  - การวางแผนติดตามผล
- บันทึกการรักษาและยืนยันว่า:
  - นัดหมายถูกเปลี่ยนเป็นสถานะเสร็จสิ้น
  - สต็อกถูกตัดอัตโนมัติ
  - ระบบแสดงข้อความสำเร็จชัดเจน

### 5. ความลึกของ Patient Profile
- เปิดหน้าโปรไฟล์ของผู้รับบริการคนเดิม
- แสดง:
  - ประวัติการรักษา
  - การมองเห็นแพทย์ผู้รับผิดชอบ
  - แท็บข้อความ
  - แท็บเฉพาะงานคลินิกที่ถูกควบคุมด้วยสิทธิ์
  - แท็บการเงินที่แสดงเฉพาะ role ที่ได้รับอนุญาต

### 6. Workflow การส่งข้อความ
- เปิดหน้า `Messages`
- สาธิต:
  - ปุ่มลัด LINE / WhatsApp / โทรออก
  - การส่งข้อความพร้อมบันทึก log
  - UI ที่เปลี่ยนตามช่องทางติดต่อที่มีจริงของผู้รับบริการ

### 7. การเงิน
- เปิดหน้า `Financial`
- เพิ่มรายการรับชำระหรือรายการค้างชำระ
- แสดง:
  - empty state แบบพรีเมียมเมื่อยังไม่มีข้อมูล
  - แท็บค้างชำระ
  - การเข้าถึงตามสิทธิ์

### 8. สินค้าและคลังสต็อก
- เปิด `Settings > Product Library`
- แสดงการแก้ไขราคา สต็อก จำนวนต่อขวด/กล่อง และค่าการแจ้งเตือนสต็อกต่ำ
- เปิด `Settings > Inventory`
- สาธิต:
  - การรับเข้า stock
  - การป้องกันไม่ให้เบิกใช้/สูญเสียจนสต็อกติดลบ
  - การปรับยอดพร้อมบันทึกหมายเหตุของส่วนต่าง
  - การมองเห็น batch / expiry
  - recent lot chips

### 9. เครื่องมือด้าน clinical compliance
- เปิด `Consent Form` ของผู้รับบริการ
- เปิด `Digital Notepad`
- อธิบายว่าหน้า clinical ที่มีความอ่อนไหวถูกป้องกันด้วยสิทธิ์ clinical แล้ว

### 10. Settings และการดูแลระบบ
- เปิดหน้า `Settings`
- แสดง:
  - profile card ของพนักงานจริง
  - การตั้งค่าระบบข้อความ
  - audit logs
  - security
  - treatment rules
- ระบุว่ารายการ placeholder ที่ยังไม่พร้อมถูกนำออกแล้วเพื่อให้การส่งมอบดูพร้อมใช้งานจริง

## หมายเหตุสำหรับผู้นำเสนอ

- เน้นว่าการ assign แพทย์เชื่อมต่อจาก appointment ไปถึง treatment แล้ว
- เน้นว่าการควบคุมสิทธิ์มีทั้งในระดับ UI visibility และ route access
- เน้นว่า inventory รองรับการใช้งานแบบจำนวนทศนิยมและมี traceability ที่ดีขึ้น
- เมื่อหน้าใดไม่มีข้อมูล ให้หยุดสั้น ๆ เพื่อโชว์ empty state ที่ถูก polish แล้ว

## เดโมเสริมตามบทบาทผู้ใช้งาน

### บทบาทแพทย์
- แสดงการเข้าถึงข้อมูล clinical และ financial
- แสดง action ที่สามารถทำได้บนผู้รับบริการและการรักษา

### บทบาทพนักงานต้อนรับ / สิทธิ์จำกัด
- แสดงข้อจำกัดในการเข้าถึง:
  - ไม่สามารถเข้าหน้าการเงินที่มีความอ่อนไหว
  - ไม่สามารถเข้าหน้าสร้าง/แก้ไขผู้รับบริการผ่าน route ตรงได้
  - ไม่สามารถใช้เครื่องมือ clinical-only

## เกณฑ์ความสำเร็จของเดโม

- สามารถสร้างผู้รับบริการใหม่พร้อม validation ที่ถูกต้องได้
- สามารถสร้าง appointment และ assign แพทย์ได้
- สามารถเปิด treatment จาก appointment และบันทึกได้สำเร็จ
- appointment เปลี่ยนเป็น completed หลังบันทึก treatment
- มีรายการ stock transaction แสดง และสต็อกไม่ติดลบ
- ขอบเขตสิทธิ์การเข้าถึงมองเห็นได้และน่าเชื่อถือ
- หน้า Settings ดูพร้อมใช้งานในระดับ production-ready
